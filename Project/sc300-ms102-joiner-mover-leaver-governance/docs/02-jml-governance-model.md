# Joiner-Mover-Leaver Governance Model

## Governance Objective
Identity lifecycle changes must be standardized, repeatable, and validated so that user access follows employment status and job function.

## Joiner Principle
New users receive baseline access through department and license groups instead of one-off manual assignment.

## Mover Principle
When a user changes departments or roles, legacy access must be removed before the access model can be considered complete.

## Leaver Principle
When a user leaves the company, sign-in must be blocked first, then licenses and resource access must be removed in a controlled order.

## Control Logic
- Group-based assignment where possible
- Least privilege by department and job function
- Repeatable evidence capture for every lifecycle event
- Clear separation between employee access and admin access
