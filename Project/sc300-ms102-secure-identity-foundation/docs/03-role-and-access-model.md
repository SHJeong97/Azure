# Role and Access Model

## Administrative Personas
- Global Administrator (break-glass / limited use)
- Identity Administrator
- User Administrator
- Helpdesk Administrator
- Exchange Administrator
- SharePoint Administrator

## Access Design Principle
Administrative access must be role-based, limited, documented, and aligned to job function.

## Department Access Pattern
- HR users receive HR collaboration access only.
- Finance users receive Finance collaboration access only.
- Sales users receive Sales collaboration access only.
- IT users receive elevated admin capability only where required.

## Risk Statement
If administrative roles and user access are assigned manually without standards, the company increases the chance of privilege creep, support inconsistency, and access retention after role changes.
