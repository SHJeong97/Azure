# Validation Plan

## What Will Be Validated
- Users created successfully
- Groups created successfully
- License assignment works through groups
- Authentication controls apply to intended users
- Admin roles are separated correctly
- Exclusions are documented and intentional

## Evidence Types
- Portal screenshots
- PowerShell output
- Group membership views
- License assignment views
- Conditional Access policy results
- Test sign-in outcomes

## Validation Standard
Every major configuration must have at least one screenshot and one written validation note.
