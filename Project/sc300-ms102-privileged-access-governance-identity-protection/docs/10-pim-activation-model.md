# PIM Activation Model

## Business Objective
Privileged access should be available when needed without remaining permanently active.

## Design Decision
The project will convert User Administrator access for Ethan Walker into a PIM-managed eligible role instead of a standing active role.

## Activation Requirements
- MFA required for activation
- justification required
- time-bound activation duration
- activation validated through portal evidence

## Role in Scope
- User Administrator

## Pilot User
- ethan.walker@democompany1016.onmicrosoft.com

## Business Value
This reduces standing privilege while preserving operational capability for user and group administration tasks.
