# Validation Plan

## What Will Be Validated
- pilot users created successfully
- pilot groups created successfully
- E5 licensing assigned correctly
- admin roles assigned correctly
- PIM eligibility configured correctly
- privileged activation requirements configured correctly
- Identity Protection and risk-based policy configuration captured
- validation evidence exists for each major control

## Evidence Types
- portal screenshots
- PowerShell / Graph output
- role membership views
- license views
- PIM assignment views
- policy configuration views
- validation notes

## Validation Standard
Every major privileged-access or identity-protection control must have at least one screenshot and one written validation note.
