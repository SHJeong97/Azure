# Risk-Based Protection Strategy

## Objective
Use Microsoft Entra identity-risk controls to improve detection and response for suspicious sign-ins and potentially compromised users.

## Planned Control Areas
- user risk policy design
- sign-in risk policy design
- protected admin access path
- stronger sign-in requirements for privileged accounts
- validation of identity-risk configuration and response flow

## Business Logic
If risky users and risky sign-ins are not handled consistently, compromised accounts may retain access longer than the business can tolerate.
